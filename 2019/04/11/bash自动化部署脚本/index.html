这是18年部署战争游戏时，写自动化部署脚本的一点经验。

感觉shell写起来和python有点像……

登录服务器

开头需要指明使用expect执行脚本，因为这些指令不是bash自带的；

使用的是spawn指令发送ssh连接请求，然后用expect表示等待对方输出后再做反馈，是一个灵活的指令；

#! /usr/bin/expect
# 如果不想明文写在脚本里，可以设置为命令行参数
set ipaddr [lindex $argv 0];
set username [lindex $argv 1];
set password [lindex $argv 2];

set timeout 30 # 超时等待时间
set prompt "#" # 这里是登陆成功的输出，linux简单用#匹配最后一个字符就可以

spawn ssh $username@$ipaddr

expect {
    "yes/no" { send "yes\r"; exp_continue } # 继续执行expect语句
    "*assword" { send "$password\r" } # 密码输入提示，忽略p大小写
    "Permission denied*" { exit } # 密码错误
    timeout { puts "Timed out during login"; exit }
}

expect "$prompt"
interact # 交还控制权

进入服务器后执行指令要用send指令。

结束时，如果把权限交还用户则需要用interact。

# ...
# 结束连接
send "exit\r"

文件指令

文件主要使用scp指令进行本地和远程的传输。它是seccure copy的缩写，和cp比较区别在于，前者可以远程传输，而且是加密的；后者只能本机移动。

以文件上传到远程服务器为例，展示具体使用格式（下一步要输入密码）：

# 复制单个文件
scp $file $username@$ipaddr:$remote_folder
# 递归复制整个目录
scp -r $folder $username@$ipaddr:$remote_folder

在脚本中，我们使用expect模块执行scp指令：

#! /usr/bin/expect
# 复制文件
spawn scp $file $username@$ipaddr:$remote_folder
expect {
  "yes/no" { send "yes\r"; exp_continue }
  "*assword" { send "$password\r" }
  timeout { exit }
}

# 等待传输结束
expect "100%"
expect eof

对文件夹，只需要把上面spawn scp $file $username…改成spawn scp -r $folder $username…即可。

启动服务

需要提前在服务器修改nginx配置，并配置好本地python环境（gunicorn+flask等等）。

这个没什么好说的，常规启动服务即可。

send "kill -9 $(pidof gunicorn)"
send "nginx -s restart && nohup gunicorn -w 1 -b 127.0.0.1:8080 main:app -D"

有关部署的细节，详见另一篇文章。

总体代码：

#! /usr/bin/expect
# 如果不想明文写在脚本里，可以设置为命令行参数
set ipaddr [lindex $argv 0];
set username [lindex $argv 1];
set password [lindex $argv 2];
set folder "./war"
set remote_folder "~/war"
set timeout 30

# 复制文件
spawn scp $file $username@$ipaddr:$remote_folder
expect {
  "yes/no" { send "yes\r"; exp_continue }
  "*assword" { send "$password\r" }
  timeout { exit }
}

# 等待传输结束
expect "100%"
expect eof

spawn ssh $username@$ipaddr

expect {
    "yes/no" { send "yes\r"; exp_continue } # 继续执行expect语句
    "*assword" { send "$password\r" } # 密码输入提示，忽略p大小写
    "Permission denied*" { exit } # 密码错误
    timeout { puts "Timed out during login"; exit }
}

expect "#"

# 运行
send "kill -9 $(pidof gunicorn)\r"
send "nginx -s restart && cd ~/war/ && nohup gunicorn -w 1 -b 127.0.0.1:8080 main:app -D\r"
