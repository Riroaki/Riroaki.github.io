Redis

接下来安装redis：

$ sudo apt install make gcc make-guile
# 将redis解压，放在usr/local文件目录下，然后进入redis目录：
$ make PREFIX=/usr/local/redis-5.0.4 MALLOC=libc install
# 将conf放在bin目录下
$ cp redis.conf bin/
$ cd bin && tree
.
├── redis-benchmark
├── redis-check-aof
├── redis-check-rdb
├── redis-cli
├── redis-sentinel -> redis-server
├── redis-server
└── redis.conf

0 directories, 7 files

启动redis：

  • 可以在bin直接启动：./redis-server，但是这样是前端启动，会话关闭就停止运行
  • 在配置文件修改守护进程选项，变成后台运行：
# /usr/local/redis-5.0.4/bin/redis.conf
################################# GENERAL #####################################

# By default Redis does not run as a daemon. Use 'yes' if you need it.
# Note that Redis will write a pid file in /var/run/redis.pid when daemonized.
daemonize yes # no就是前台运行，后台运行就要改成yes

运行：

# /usr/local/redis-5.0.4/bin/
$ ./redis-server ./redis.conf

10442:C 24 Apr 2019 13:36:47.716 # oO0OoO0OoO0Oo Redis is starting oO0OoO0OoO0Oo
10442:C 24 Apr 2019 13:36:47.716 # Redis version=5.0.4, bits=64, commit=00000000, modified=0, pid=10442, just started
10442:C 24 Apr 2019 13:36:47.716 # Configuration loaded

启动客户端进行连接：

$ ./redis-cli 
127.0.0.1:6379>

关闭（shutdown则数据不会丢失）：

$ ./bin/redis-cli shutdown
$ ./bin/redis-cli shutdown # 强行关闭，有风险

配置执行文件：

# .../redis-5.0.4/
$ sudo cp bin/redis-server /usr/local/bin
$ sudo cp bin/redis-cli /usr/local/bin

搭建redis集群

# .../redis.5.0.4/
$ mkdir redis_cluster
$ cd redis_cluster
$ mkdir 7000 7001 7002             #redis集群要求最少有3个节点
$ touch 7000/redis.conf
$ touch 7001/redis.conf
$ touch 7002/redis.conf

对刚才创建的三个文件进行修改：

port 7000               # 端口7000,7001,7002，与目录对应
bind 127.0.0.1    #默认ip为127.0.0.1，需要改为其他节点机器可访问的ip，
                                    #否则创建集群时无法访问对应的端口，无法创建集群
daemonize yes       #redis后台运行
cluster-enabled yes                                      #开启集群
cluster-config-file nodes_7000.conf      #集群的配置，配置文件首次启动自动生成 7000，7001，7002  
cluster-node-timeout 8000                       #请求超时，默认15秒，可自行设置
appendonly yes      #开启aof持久化模式，每次写操作请求都追加到appendonly.aof文件中
appendfsync always                                      #每次有写操作的时候都同步
logfile "/data/redis/logs/redis.log"     #redis服务日志
pidfile /var/run/redis_7000.pid              #pidfile文件对应7000，7001，7002

启动节点：

# .../redis.5.0.4/bin/
$ ./redis_server ../redis_cluster/7000/redis.conf
$ ./redis_server ../redis_cluster/7001/redis.conf
$ ./redis_server ../redis_cluster/7002/redis.conf

检查运行状态：ps -ef | grep redis

在多个服务器搭建多个集群

同上操作，对其他服务器做一样的配置即可。

节点启动和关闭脚本

如果有多个节点可以这么启动和关闭：

# 启动，在每个服务器执行
for((i=0;i<3;i++)); do redis-server ~ redis-5.0.4 redis-cluster 700$i redis.conf; done # 关闭，ip为三台服务器对应ip for((i="0;i<3;i++));" redis-cli -c -h $ip -p shutdown;

无法启动

删除每个集群的配置文件（第一次运行会产生的nodes_700x.conf），再对每个服务器重启
